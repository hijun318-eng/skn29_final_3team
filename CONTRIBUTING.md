# Contributing

이 저장소 또는 이 협업환경으로 생성한 새 프로젝트에 기여하기 전에 다음 문서를 순서대로 확인한다.

1. [팀원 개발환경 세팅 가이드](./docs/engineering/TEAM_DEVELOPMENT_SETUP_GUIDE.md)
2. [Git branch 전략](./docs/engineering/GIT_BRANCH_STRATEGY.md)
3. [최소 프로젝트 폴더 구조](./docs/engineering/PROJECT_STRUCTURE.md)
4. [Commit convention](./docs/engineering/COMMIT_CONVENTION.md)
5. [기능별 품질 평가 가이드](./docs/engineering/QUALITY_EVALUATION_GUIDE.md)
6. [Codex 개발환경·하네스](./docs/engineering/CODEX_HARNESS.md)

기본 작업 흐름은 `Issue -> 개인 branch -> 검증 -> PR(dev) -> 통합 검증 -> PR(main)`이다. 코드와 개발 문서를 포함한 모든 변경은 `junhee`, `minji`, `seung`, `daesung`, `jaehong` 중 담당자의 branch에서 수행하고 `main`과 `dev`에서는 직접 수정하거나 push하지 않는다.
