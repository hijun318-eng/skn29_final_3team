# 요구사항 정의서 — Walkerhill VOC Operations Intelligence Agent (v2)

**SK네트웍스 Family AI 캠프 29기 3팀** · 공식 서비스명 **SensePlace** · 호텔 VOC·운영 이슈 분석 에이전트

> VOC와 운영 데이터를 동일 시설·기간 기준 교차분석하여 원인 후보·근거·우선 점검·검토 대응 옵션을 제공하는 내부 운영자용 서비스. 반응형 챗봇이 아닌 **감시→조사→리포트→대응 옵션 추천** 자율 에이전트이며, AI는 원인을 확정하지 않고 자동 실행하지 않는다(최종 조치=운영자 승인).
> 작성 기준일 2026-07-20 · v2.1 · 데이터: 합성 운영 데이터+합성 VOC 전용 · 정식 요구사항 **62개**

> 제품·Baseline 기준은 `docs/markdown/SensePlace_기획서_초안.md`를 따른다. 기존 62개 ID는 추적성을 위해 보존하고 아래 활성 범위 분류가 기존 `MVP` 표기보다 우선한다. 화면설계서는 이번 동기화 범위에서 제외한다.

## 📌 핵심 원칙

1. 핵심은 **VOC×운영지표 교차분석**(단순 감성분석 아님).
2. **수치 계산과 이상 판정은 SQL·Python·규칙, LLM은 질문 해석·근거 기반 설명**만 담당한다.
3. **PostgreSQL `metric_catalog` 기반 semantic catalog로 지표·허용 grain·권한을 고정**한다. VectorDB는 독립 실험 전용이다.
4. 모든 결과에 **근거 수치·기간·리뷰** 첨부, 원인은 **후보**로만 표현.
5. 개인정보 저장 전 마스킹, 최종 결정·조치는 운영자.

```
전량 합성 데이터 생성→품질 Gate→PostgreSQL 적재
  ├─ 기능 A: Django session·권한→보장 질문 8종→SQL Guard→표·차트·설명
  └─ 기능 B: 결정론적 감지→Incident LangGraph→근거·반대 근거→보고서 DRAFT→승인·보류·반려
```
