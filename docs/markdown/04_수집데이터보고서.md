## 1. 수집 데이터 개요

SensePlace 프로토타입은 개인정보가 없는 합성 VOC와 객실·조식·인력 운영지표를 같은 서비스 영역과 기간으로 연결한다. 이 문서는 fixture 생성 전의 사전 설계 초안이며, 실제 워커힐 데이터는 사용하지 않는다.

| 데이터명 | 수집 대상 | 수집 목적 | 사용 예정 기능 | 출처 / 저작권 |
|---|---|---|---|---|
| 합성 VOC source fixture | 가상 VOC 문장·생성 lineage | VOC 입력·재현 | 정규화·분류·검증 | 팀 자체 생성 |
| 검증 VOC·주제 | VOC·복수 주제·감성·근거 | 부정 주제·근거 표시 | VOC 추세·evidence | 팀 자체 생성·가공 |
| 객실 일별 운영 | 판매가능·판매·점유 객실 수 | 객실 수요·VOC 비교 | 점유율·VOC 교차분석 | 팀 자체 합성 |
| 조식 15분·일별 운영 | 도착·처리·대기열·p90 | 시간대별 혼잡 비교 | 피크·일별 p90 분석 | 팀 자체 합성 |
| 인력 shift | 계획·실제·결원 인원·근로분 | 수요·인력 균형 비교 | 15분 bucket·shift 연결 | 팀 자체 합성 |
| manifest·metric·role | dataset·지표·역할 정의 | 재현·계산·조회 범위 고정 | 품질 Gate·지표·권한 | 팀 자체 정의 |

## 2. 수집 방법 및 자동화 절차

### 2.1 수집 방식

| 수집 방식 | 선택 | 수집 방식 | 선택 |
|---|---|---|---|
| 웹 크롤링 | 미선택 | API 호출 | 미선택 |
| 팀 자체 합성 데이터 생성 | 선택 | 외부 실데이터 수집 | 미선택 |

### 2.2 수집 환경·자동화·예외 처리

| 항목 | 내용 |
|---|---|
| 사용 언어 / 라이브러리 | Python 3.10+ / 생성 구현 시 프로젝트 dependency로 확정 |
| 자동화 여부 및 주기 | versioned schema·generator config·고정 seed batch. 최초 생성, schema·scenario·generator version 변경, 회귀검증 필요 시 실행 |
| 오류 발생 시 예외 처리 전략 | schema·PII·중복·FK·범위 검사 중 하나라도 실패하면 batch 전체를 저장하지 않고 실패 사유를 기록 |

### 2.3 예시 스크립트 또는 흐름도

```text
schema·지표·시간 계약 정의
→ 고정 seed·scenario 선택
→ 합성 VOC·운영 데이터 생성
→ schema·PII·중복·FK·범위 검사
→ 검증 VOC 정규화·주제 연결
→ Parquet/JSONL 및 PostgreSQL 적재 후보 생성
→ manifest에 version·seed·기간·품질 상태 기록
```

### 3.2 데이터 양

| 항목 | 값 |
|---|---|
| 전체 수집 데이터 건수 | 0건 — fixture 생성 전 설계 단계 |
| 추출된 고품질 데이터 건수 (필터링 후 기준) | 0건 — 생성·품질 검증 전 |

실제 생성 후에는 `dataset_manifest`와 품질 검사 결과를 근거로 위 건수를 갱신하고, 후속 데이터전처리결과서의 원천 데이터 건수와 일치하는지 확인한다.

## 5. 데이터 품질 및 정합성 관리 방안

| 항목 | 관리 방안 |
|---|---|
| 중복 제거 기준 | VOC는 `voc_id`, 주제는 `(voc_id, topic_code)`, 운영 fact는 dataset·service_area·영업일/구간 복합키로 중복 차단 |
| 정합성 검증 방법 | schema type·필수값, manifest·service area FK, 시각 순서, 15분 경계, shift 구간, 수치 범위, VOC 근거 offset, PII pattern 검사 |
| Null 처리 및 결측치 전략 | `occurred_at`은 확인 불가 시만 NULL 허용하고 `RECEIVED_ONLY`로 표시. 필수 키·version·지표 결측 batch는 저장 거부 |
| 표준화 전략 | timestamp UTC 저장, 영업일 Asia/Seoul 해석, 코드 allowlist, UTF-8, snake_case, `metric_catalog` 단위 사용 |
| p90 계산 | 조식 일별 p90은 15분 p90의 평균이 아니라 같은 원천 대기시간 표본에서 직접 산출 |
| 적재 기준 | pre-load 검사를 통과해 `dataset_manifest.quality_status=PASSED`인 dataset만 분석 테이블에 연결 |
