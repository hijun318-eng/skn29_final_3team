# 요구사항 정의서 — Walkerhill VOC Operations Intelligence Agent (v2)

**SK네트웍스 Family AI 캠프 29기 3팀** · 호텔 VOC·운영 이슈 분석 **자율 분석 에이전트**

> VOC와 운영 데이터를 동일 시설·기간 기준 교차분석하여 원인 후보·근거·우선 점검·검토 대응 옵션을 제공하는 내부 운영자용 서비스. 반응형 챗봇이 아닌 **감시→조사→리포트→대응 옵션 추천** 자율 에이전트이며, AI는 원인을 확정하지 않고 자동 실행하지 않는다(최종 조치=운영자 승인).
> 작성 기준일 2026-07-16 · v2.0 · 데이터: 공개 리뷰(컬럼 참고)+자체 합성 · 정식 요구사항 **62개**

## 📌 핵심 원칙

1. 핵심은 **VOC×운영지표 교차분석**(단순 감성분석 아님).
2. **수치 계산은 SQL·Pandas·규칙, LLM은 분류·질문해석·설명**. 원인 후보는 규칙으로.
3. **온톨로지(시맨틱 레이어)로 지표 정의를 못 박아 text-to-SQL 정확도 보장**. 벡터DB는 근거 리뷰 검색 전용.
4. 모든 결과에 **근거 수치·기간·리뷰** 첨부, 원인은 **후보**로만 표현.
5. 개인정보 저장 전 마스킹, 최종 결정·조치는 운영자.

## 🎯 목표와 비목표

**목표:** 공개 리뷰+합성 운영 데이터로, 동일 시설·기간 VOC×운영지표를 교차분석해 이상징후→원인 후보·근거·대응 옵션을 제공하는 자율 분석 에이전트를 8주 내 MVP로 구현.

**비목표:** 인과 확정 · 조치 자동 실행 · 실시간 고객 응대 · 완전 자유형 BI · 실제 PMS·POS·CRM 연동 · 상용 다중 호텔 운영.

### MVP 핵심 완료 흐름
```
합성 데이터 파이프라인(스키마 정의서→생성→검증→정답표)
  → 온톨로지(지표 정의)  → 감성·주제 분류(근거)
  → [트리거] 이상징후 자동 감지  → 에이전트 조사(교차분석 + text-to-SQL)
  → 원인 후보 + 근거(수치·리뷰)  → 검토 대응 옵션
  → 대시보드 / 주간 리포트  → (eval 정답표로 정확도 자동 채점)
```
