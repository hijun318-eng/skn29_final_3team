## 1. 배경과 문제
호텔 VOC(리뷰·설문)는 "무엇을 불편해했는지"는 보여주지만 **왜 그 시점에 불만이 늘었는지**는 운영 데이터와 분리돼 설명하지 못한다. 담당자는 리뷰와 운영 시트를 손으로 대조하고, "무엇을 물어야 할지" 몰라 방치하며, 분석가 병목에 걸린다.

## 2. 목표와 비목표
- **목표:** 동일 시설·기간 VOC×운영지표 교차분석으로 이상징후→원인 후보·근거·검토 대응 옵션을 제공하는 **자율 분석 에이전트**를 8주 MVP로 구현.
- **비목표:** 인과 확정 · 조치 자동 실행 · 실시간 고객 응대 · 완전 자유형 BI · 실제 PMS·POS·CRM 연동 · 상용 다중 호텔.

## 3. 페인포인트 (누가·무엇을)

> 고통받는 사람은 **호텔 고객이 아니라 내부 운영자**다.

| 페르소나 | 현재(As-Is) 고충 | 우리가 없애는 것 | 관련 요구사항 |
|---|---|---|---|
| 고객경험 담당자(실무자) | 리뷰·운영 시트 수작업 대조(수시간) | VOC×운영 자동 교차분석 | AI-008, FUN-006 |
| F&B·객실 팀장(실무자) | 대기·혼잡 불만과 인력·방문객 연결 안 됨 | 동기간 운영지표 자동 매칭 | AI-005~008 |
| 경영진(관리자) | 주간 보고 수작업, 감으로 우선순위 | 근거 포함 주간 리포트 자동 | RPT-001, UI-002 |
| 전체 | 문제를 뒤늦게 발견(사후 대응) | 이상징후 트리거로 조기 감지 | FUN-013, AI-006 |

핵심 pain 한 줄: **"데이터는 있는데 무엇을 물을지 몰라 방치되고, 원인 파악이 분석가 병목에 걸린다."** (합성 데이터 기반 제안 페르소나 — 가정임을 명시)

## 5. 서비스 범위
- **MVP 필수:** 합성 파이프라인·온톨로지·에이전트 코어+트리거·text-to-SQL(시각화)·감성/주제 분류·이상탐지·교차분석·원인 후보·근거 리뷰·대시보드·주간 리포트·eval·Level2 인증
- **MVP 제외:** 실시간 PMS·POS 연동·고객 자동응답·자동 조치·인과 자동 판정·모바일·대규모 다국어·자체 LLM 사전학습
- **stretch:** MCP 읽기·전달(비즈니스 연결) / **발전가능성:** MCP 쓰기·GraphRAG·메모리·예측·다국어

## 6. 제품 형태 — 챗봇이 아닌 자율 에이전트
반응형 Q&A가 아니라 **목표·트리거를 받아 스스로 감시→조사→리포트→대응 옵션 추천**하는 능동형. text-to-SQL은 화면 주인공이 아니라 **에이전트가 조사 때 호출하는 도구**이며, 실행(act)은 하지 않고 운영자 승인까지만 간다(안전장치=차별점).

## 7. 아키텍처 · 기술 스택

| 영역 | 담당 |
|---|---|
| 수치 계산(증감률·비율·이동평균) | **SQL / Pandas / 규칙** (비-AI) |
| 이상징후 | 규칙 + 비지도(Isolation Forest) |
| 감성·주제 분류, 질문 해석, 설명 | **LLM / sLLM** |
| 지표 정의·text-to-SQL 그라운딩 | **온톨로지/시맨틱 레이어** |
| 원인 후보 판정 | 사전 정의 규칙 |
| 최종 결정·조치 | **운영자** |

스택: **React+Tremor** / **Django+DRF**(웹·인증 Level2·관리자·게이트웨이) / **FastAPI**(AI) / **LangGraph**(supervisor·트리거) / **PostgreSQL+pgvector** / **text-to-SQL**(sqlglot 검증·read-only) / **eval+Langfuse**. 하이브리드 백엔드(Django↔FastAPI)는 인증·관리자는 Django, AI·에이전트는 FastAPI로 분리하고 Django가 게이트웨이 역할.

## 8. 데이터 계획
공개 리뷰(Kaggle·공개 데이터셋·Google Places API 등)는 **컬럼·스키마 참고용**, 실제 데이터는 **합리적 기준으로 자체 생성(합성)**. 파이프라인: 스키마 정의서(온톨로지 초안)→규칙+통계분포 생성(Faker/SDV)→결측·이상치 주입→전처리→분포·상관 검증→정답표. 합성은 `synthetic`·seed·schema_version 기록, 실제처럼 표현 금지.
