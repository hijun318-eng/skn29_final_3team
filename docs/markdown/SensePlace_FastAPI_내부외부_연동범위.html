SensePlace — FastAPI 내부·외부 연동 범위

호텔 VOC·운영 지원 플랫폼 백엔드 연동 경계 기준 · 구현 전 검토 초안(2026-07-21)

0. 결론

FastAPI는 외부 공개 API가 아니라 내부 AI·분석 서비스로 제한한다.
Browser → Django 공개 API → Django Worker → FastAPI 내부 API → PostgreSQL 또는 외부 LLM API

Django는 인증·권한·job·report·decision의 원본이며, FastAPI는 품질 검사·분석·SQL Guard·Incident·LLM 연동을 담당한다.

0.1 사람이 판단해야 할 사항

  • ☐ Django와 FastAPI 사이 내부 인증을 service token으로 할지 결정
  • ☐ FastAPI가 PostgreSQL 분석 View를 직접 조회할지 결정
  • ☐ LLM 기본 모델·대체 모델·timeout 값을 결정
  • ☐ Django가 FastAPI를 동기 호출할지 worker 기반 비동기로 처리할지 결정
  • ☐ VectorDB 실험을 FastAPI 내부에 둘지 별도 실험 코드로 분리할지 결정

0.3 최소 기능 구현 방향

P0 Django가 공개 API와 상태 관리를 담당하고, FastAPI는 내부 분석 API만 제공한다.
Django 로그인·권한 → job 생성 → Django Worker → FastAPI query/incident 실행 → PostgreSQL read-only 조회 → 선택적 LLM 1회 → 구조화 결과 반환 → Django 저장·polling·승인
