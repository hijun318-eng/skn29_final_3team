# VOC 핵심 개선사항 요약

> 2026-07-24 압축본. 상세 논문 검토와 페이지 근거는 `legacy/VOC_핵심개선사항_분석_원본_20260724_1138.md`에 보관한다.

## 적용 가능한 방법

- VOC를 감성뿐 아니라 서비스 영역·주제·긴급도·요구로 다중 분류한다.
- 유사 이슈를 묶어 최초 발생, 재발, 조치 전후 관찰 창을 관리한다.
- 원인 후보마다 VOC 문장, 운영 수치, 반대 근거와 추가 확인 과제를 연결한다.
- 관리자 승인·보류·반려, 담당 역할, 기한과 조치 이력을 남긴다.
- 빈도·심각도·처리시간·재발률을 함께 사용해 개선 우선순위를 검토한다.

## 방법론 사용 경계

SERVQUAL, Kano, QFD와 과거 호텔 사례는 분류·검토 방법으로만 참고한다. 특정 연구의 빈도, 순위, 품질 유형과 개선 효과를 워커힐의 사실이나 우선순위로 전용하지 않는다.

## 평가 기준

- 분류·검색·탐지 결과가 같은 입력과 version에서 재현되는가
- 근거와 반대 근거가 결과에 연결되는가
- 담당자가 제안 내용을 수정하거나 거부할 수 있는가
- 조치 후 동일 이슈의 재발 여부를 추적할 수 있는가
- 합성 결과를 실제 운영 성과로 표현하지 않는가

공식 기능 범위와 수용 기준은 `../01_요구사항정의서.md`를 우선한다.
