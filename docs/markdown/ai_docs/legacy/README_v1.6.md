## 폴더 역할

`docs/markdown/ai_docs/`는 공식 번호 산출물과 분리한 참고 자료 보관 위치다. 현재 참고문서는 폴더 루트에 두고 수정·삭제 전 원본은 `legacy/`에 분리한다.

- AI가 작성한 기획·검토·계약 보조 자료
- 워커힐·호텔 VOC 관련 외부 조사와 분석
- 공식 일정·교육 운영의 조회 시점 스냅샷
- 과거 통제 문서와 저장소 구조 감사 자료
- 수정·삭제 전 원본을 보존하는 `legacy/`

## 사용 원칙

- 현재 요구사항·범위·수용 기준은 `../01_요구사항정의서.md`를 우선한다.
- 일정·담당·상태는 `../02_WBS.md`, 화면 상세는 `../05_화면설계서.md`를 우선한다.
- 현재 구현 여부와 성능은 실제 코드·테스트·데이터에서 확인한다.
- 이 폴더의 파일명·경로·링크와 참고 작성 과정은 공식 제출본에 포함하지 않는다.
- 법령·라이선스·저작권·데이터 출처처럼 필요한 출처 표시는 제출 요건에 맞게 유지한다.
- 과거 경로와 표현이 남은 스냅샷은 현재 저장소 구조나 확정 사실로 해석하지 않는다.

## 문서 정리와 신규 문서 검토

다음 두 문서는 삭제·병합·요약 대상에서 제외한다.

- `최종_프로젝트_산출물_및_전체_일정.md`
- `최종_프로젝트_오프닝_자료_요약_3팀.md`

그 밖의 문서는 필요에 따라 삭제하거나 다른 문서에 병합·요약할 수 있다.

`ai_docs/`에 문서 파일이 추가될 때마다 다음 검토를 진행한다.

1. 신규 문서를 위 두 문서와 기존 기준 문서에 대조한다.
2. 내용이 상충하면 문서 기준일이 가장 최신인 문서의 내용을 보존한다.
3. 상충 내용을 아래 표에 50자 이내로 기록한다.
4. 기록 후 정리 제외 문서는 유지하고, 그 밖의 오래된 상충 내용을 삭제한다.

## 원본 문서 보관

문서를 수정하거나 삭제하기 전에 원본을 `legacy/`에 보관한다.

- 수정: 수정 전 원본의 복사본을 `legacy/`에 생성한 뒤 현재 경로의 문서를 수정한다.
- 삭제: 삭제할 원본을 `legacy/`로 이동하고 현재 경로에서 제거한다.
- 파일명: 원본 파일명에 version을 붙인다. version이 없거나 이름이 겹치면 `_원본_YYYYMMDD_HHMM`을 추가한다.
- 보존: `legacy/`의 파일은 수정·삭제·병합·요약하지 않는다.
- 사용: `legacy/`는 과거 이력 전용이며 신규 문서 상충 검토와 현재 기준 문서에서 제외한다.
