# 회의록 정리

## 1. 고객 데이터 — 예약 연동 VOC 수집 링크 [구현]

- 예약 정보가 담긴 **개별 링크**를 고객에게 제공 (random_guest_id, random_membership_id 토큰 기반)
- 고객은 로그인 없이 링크로 접속해 **VOC를 직접 전달** → 예약·고객 정보와 매핑되어 저장 → 모니터링 화면에 표시
- 전제 조건: 토큰은 추측 불가 + 만료 처리(링크 유출 방지), 고객 입력값 정제(길이 제한·스크립트 차단)

## 2. 로그인 — 역할 기반 [구현]

- role 기반 설계. 여기서 만든 role을 3번 권한 분리, 5번 보고서 자동 매핑에 그대로 재사용

## 3. 실시간 모니터링 [구현]

**핵심 (먼저)**

- 맵 확장 + 지도에서 VOC 발생 시 팝업
- 심각도별 알림: 정상 = 알림 없음 / 주의·위험 = 알림
- 운영 시간 기준으로 알림 on/off (처음엔 전역 → 이후 장소별)
- 현장 대응 진행 표시(진행중/완료 등) 시각화

**정적 / 보류**

- 근무표(정적)만 표시 — **실시간 위치추적은 제외**
- 주간/야간 모드 — *테마인지 권한 분리인지 정의 필요(미정)*
- 실시간 동작 인지(해/달/비 동적 요소) — **추후 추가, 필수 아님**

**점진 구현 (VOC 대응 절차)**

- 순서: 진행결과 표시 → 담당부서·연락처(정적) → 지원핑(상태 플래그) → 절차 매뉴얼

## 4. 대화형 에이전트 [구현]

- WISE 벤치마킹
- **text-to-SQL → 데이터 + 시각화** (자연어 질문 → 조회 → 표·차트)
- 방식: **검증된 쿼리에 매핑(방식 B)으로 시작**, 자유 생성(방식 A)은 확장 단계에서
  - LLM은 SQL을 창작하지 않고, 사람이 검증한 쿼리에 값만 채움 → 정확도 보장, "LLM은 결과만 표현" 철학과 일치
- 안전 가드(설계로 해결): read-only 전용 DB 계정, role 스코프 실행, statement timeout + LIMIT, 시스템 테이블·다중 statement 차단

## 5. 보고서 작성 [구현]

- **빈 템플릿으로 시작** → 자동 생성 + 사용자 수정/추가
- 권한(role)에 따라 업무 영역 자동 매핑 → 템플릿 선택 → 불러오기 → 자동 작성 → 갱신/확정
- **블록형 에디터**: text-to-SQL로 나온 차트/그래프를 보고서에 **드래그앤드롭으로 추가**
  - 자유 좌표가 아니라 **순서 기반 배치**, 위치 가이드 박스(점선) + 튜토리얼
  - Canva식 디자인 요소·복잡한 편집 기능은 **넣지 않음**

## 6. 외부 리뷰 관리자 알림 [확장]

- **수집 · 분석 · 리포팅 3개 에이전트** 구조 (심각도·성격에 따라 구분)
- 개발 순서: 수집 → 분석 → 리포팅 순차 (앞 단계 출력이 뒤 단계 입력)
- "1개 소스, 1개 트리거"로 개념검증 후 확장

## 우선순위 (단계)

- **1순위(핵심 데모 흐름)**: 로그인 → 예약 VOC 링크 → 모니터링(맵·팝업·심각도·진행표시) → 빈 템플릿 보고서
- **2순위**: 가드형 text-to-SQL(방식 B) + 차트 시각화 → 블록형 에디터로 보고서에 얹기
- **3순위(확장)**: 자유형 text-to-SQL(방식 A), 외부 리뷰 3에이전트, 실시간 동작 연출, 장소별 알림 세분화

## 아직 정할 것

- 주간/야간 모드가 "화면 테마"인지 "권한 분리"인지
- text-to-SQL 최종 방식 (B로 시작 확정 / A 확장 시점)
