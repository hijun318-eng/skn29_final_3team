# Codex 일일보고 갱신 규칙

Codex가 저장소 파일을 생성·수정·삭제한 작업을 완료하면 현재 개인 branch의 `일일보고.md`에 실제 수행 결과를 한 줄 이상 기록한다.

## Branch와 보고 파일

| Branch | 보고 파일 |
|---|---|
| `junhee` | `docs/markdown/daily_reports/junhee/일일보고.md` |
| `minji` | `docs/markdown/daily_reports/minji/일일보고.md` |
| `seung` | `docs/markdown/daily_reports/seung/일일보고.md` |
| `daesung` | `docs/markdown/daily_reports/daesung/일일보고.md` |
| `jaehong` | `docs/markdown/daily_reports/jaehong/일일보고.md` |

`main`, `dev` 또는 위 표에 없는 branch에서는 작성자를 임의로 정하지 않고 일일보고를 수정하지 않는다.

## 작성 방법

1. 현재 날짜를 KST 기준 `YYYYMMDD`로 구한다.
2. 같은 날짜의 `## YYYYMMDD` 제목이 있으면 그 목록 끝에 수행 내용을 추가한다.
3. 같은 날짜가 없으면 파일 상단 안내문 바로 아래에 새 날짜 블록을 추가한다.
4. 사용자가 이해할 수 있는 완료 결과를 간단한 과거형 또는 명사형으로 기록한다.
5. 기존 보고를 수정하거나 중복 항목을 추가하지 않는다.
6. 일일보고 작성 후 프로젝트 일정에서 해당 주차를 확인하고 아래 `주간보고 작성 규칙`에 따라 같은 주차의 보고를 갱신한다.

```markdown
## YYYYMMDD

- 수행한 작업 또는 완료 결과
```

## 기록 대상

- 코드, 문서, 설정 또는 테스트 파일을 변경한 작업
- 구조 변경, 자동화 추가, 버그 수정 등 저장소에 결과가 남는 작업

다음 작업은 기록하지 않는다.

- 파일 변경이 없는 조사·질문 답변
- commit message 초안 작성처럼 저장소를 변경하지 않은 작업
- 일일보고 누락을 보완하기 위해 일일보고 파일만 수정한 작업

## 주간보고 작성 규칙

일일보고가 추가·수정되거나 사용자가 날짜 또는 기간과 함께 주간보고 작성을 요청하면 다음 절차로 `docs/markdown/daily_reports/주간보고.md`를 갱신한다.

1. `junhee`, `minji`, `seung`, `daesung`, `jaehong`의 일일보고를 모두 읽는다.
2. `시작일~종료일`이 지정되면 양 끝 날짜를 포함한다.
3. 날짜 하나만 지정되면 해당 날짜를 종료일로 보고, 종료일을 포함한 최근 7일을 대상으로 한다.
4. `docs/markdown/최종_프로젝트_산출물_및_전체_일정.md`의 `주차 구분`을 확인해 날짜 제목 옆에 프로젝트 주차를 표시한다.
5. 시작일과 종료일이 같은 주차이면 `(N주차)`, 여러 주차에 걸치면 `(N~M주차)`로 표시한다.
6. 대상 기간의 작업을 기획, 데이터, 모델링, UI/UX, 개발환경, 협업, 멘토링 등 실제 내용에 맞는 상위 작업으로 묶는다.
7. 같은 주차의 기존 보고가 있으면 새 블록을 만들지 않고 해당 블록을 갱신한다.
8. 비슷한 작업은 하나의 상위 항목으로 합치고, 반복되는 설명은 대표 결과 한 번만 남긴다.
9. 일일보고에 없는 사실, 완료 여부, 일정 또는 담당자는 추정하지 않는다.
10. 작성자를 사용자가 지정하지 않으면 `3팀`으로 기록한다.
11. 최신 주간보고를 파일 상단 안내문 바로 아래에 추가하고 기존 보고는 보존한다.
12. 제목부터 다음 주차 제목 직전까지의 주간보고 블록은 빈 줄을 포함해 최대 50줄로 제한한다.
13. 50줄을 넘으면 핵심 결과, 결정, 이슈와 다음 작업을 우선해 다시 요약하고 세부 조사 목록과 반복 설명을 줄인다.

주간보고는 다음 형식을 사용한다.

```markdown
## YYYYMMDD ~ YYYYMMDD (N주차)

작성자: 3팀

- 상위 작업
  - 세부 수행 내용
  - 세부 수행 내용
- 멘토링 진행 (YY.MM.DD HH:mm ~ HH:mm)
  - 피드백 및 후속 작업
```

기간 내 보고가 없는 팀원이 있더라도 빈 항목을 만들지 않는다. 전체 팀원의 보고가 없으면 `- 해당 기간 일일보고 없음`으로 기록한다.
