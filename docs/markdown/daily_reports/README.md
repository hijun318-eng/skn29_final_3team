## 주차별 팀 요약·주간보고

`docs/markdown/ai_docs/최종_프로젝트_산출물_및_전체_일정.md`의 `주차 구분`으로 날짜의 `N주차`를 정하고 다음 구조로 저장한다.

```text
team_summaries/
└─ N주차/
   ├─ YYYYMMDD.md
   └─ 주간보고.md
```

- 팀원 5명의 같은 날짜 일일보고는 `team_summaries/N주차/YYYYMMDD.md`에 통합한다.
- 날짜별 파일에는 팀 공통 요약과 팀원별 요약을 두고, 날짜 블록이 없는 팀원은 추정 없이 `보고 없음`으로 표시한다.
- 각 주차 폴더에는 그 주차의 `주간보고.md`를 하나만 두며 다른 주차 내용을 함께 기록하지 않는다.
- 주간보고는 계속 개인 일일보고 5개를 직접 근거로 작성하며 날짜별 팀 요약을 원본 대신 사용하지 않는다.

## 작성 방법

1. 현재 날짜를 KST 기준 `YYYYMMDD`로 구한다.
2. 같은 날짜의 `## YYYYMMDD` 제목이 있으면 그 목록 끝에 수행 내용을 추가한다.
3. 같은 날짜가 없으면 파일 상단 안내문 바로 아래에 새 날짜 블록을 추가한다.
4. 사용자가 이해할 수 있는 완료 결과를 간단한 과거형 또는 명사형으로 기록한다.
5. 기존 보고를 수정하거나 중복 항목을 추가하지 않는다.
6. 제목부터 다음 날짜 제목 직전까지 빈 줄을 포함해 최대 5줄로 작성하고, 초과하면 관련 작업을 한 줄로 통합한다.
7. 개인 branch 작업에서는 개인 일일보고까지만 갱신한다. 팀 요약·주간보고는 사용자가 날짜·기간을 지정해 별도로 요청하거나 아래 `dev 병합 요청 시 보고 통합` 절차가 실행될 때 갱신한다.

```markdown
## YYYYMMDD

- 수행한 작업 또는 완료 결과
```

## 기록 대상

- 코드, 문서, 설정 또는 테스트 파일을 변경한 작업
- 구조 변경, 자동화 추가, 버그 수정 등 저장소에 결과가 남는 작업

다음 작업은 기록하지 않는다.

- 파일 변경이 없는 조사·질문 답변
- commit message 초안 작성처럼 저장소를 변경하지 않은 작업
- 개인 일일보고 누락이나 날짜별 팀 요약·주간보고만 보완한 보고 전용 작업

## dev 병합 요청 시 보고 통합

`merge-branch-to-dev`가 개인 branch를 `dev`에 병합한 직후 `.agents/skills/update-project-reports/SKILL.md`의 post-merge mode를 적용한다. 대상 날짜는 사용자 지정 범위를 우선하고, 지정이 없으면 병합으로 바뀐 개인보고 날짜와 아직 팀 요약이 없는 날짜를 사용한다. Skill은 변경된 `team_summaries/` 경로와 검증 결과만 Git 통합 절차에 반환하며 보고 통합 자체를 다른 보고나 WBS에 기록하지 않는다.

## 주간보고 작성 규칙

사용자가 날짜 또는 기간과 함께 주간보고 작성을 요청하거나 `dev` 병합 후 보고를 통합하면 `.agents/skills/update-project-reports/SKILL.md`를 적용한다.

- 팀원 5명의 개인 일일보고를 직접 근거로 사용한다.
- 단일 날짜 요청은 해당 날짜를 포함한 최근 7일, 기간 요청은 양 끝 날짜를 포함한다.
- 공식 일정의 주차 구분을 사용하며 여러 주차는 주차별 파일로 나눈다.
- 사용자 지정 기간은 해당 주차 안의 지정 날짜만, 병합 후 통합은 영향 주차의 보고 존재 날짜 전체를 누적한다.
- 비슷한 작업을 상위 항목으로 합치고 Git 운영 이력과 추정 사실은 제외한다.
- 작성자 기본값은 `3팀`이며 각 주간보고는 최대 40줄로 제한한다.

주간보고는 다음 형식을 사용한다.

```markdown
# N주차 주간보고

## YYYYMMDD ~ YYYYMMDD (N주차)

작성자: 3팀

- 상위 작업
  - 세부 수행 내용
  - 세부 수행 내용
- 멘토링 진행 (YY.MM.DD HH:mm ~ HH:mm)
  - 피드백 및 후속 작업
```

기간 내 보고가 없는 팀원이 있더라도 빈 항목을 만들지 않는다. 전체 팀원의 보고가 없으면 `- 해당 기간 일일보고 없음`으로 기록한다.
