## 20260720

- **SensePlace 프로젝트 방향 정리**
    - 호텔 고객 리뷰와 내부 운영데이터를 연결해 반복 문제와 원인 후보를 분석하고 관리자에게 개선방안과 리포트를 제공하는 서비스로 정의
    - `고객 리뷰·VOC 수집 → 운영데이터 교차분석 → 원인 후보 도출 → 개선안 제시 → 관리자 검토·승인 → 현장 조치 → 효과 측정` 흐름으로 정리
- **VOC·데이터 연계 및 AI 역할 학습**
    - VOC를 리뷰뿐 아니라 문의·요청·불만·설문을 포함한 고객 의견 데이터로 정의하고, 외부 리뷰는 익명 데이터로 분석
    - 시설·날짜·시간대·문제 유형을 기준으로 리뷰와 운영데이터를 연결하며, 원천 데이터 변경 시 분석 결과와 관리자 리포트가 갱신되도록 설계
    - AI는 원인 확정이나 직접 조치 대신 근거가 포함된 원인 후보와 개선안을 제안하고 관리자가 검토·승인하는 방식으로 역할 구분
    - Google Places API에서 리뷰 본문·별점·작성 시각·언어·작성자 표시 이름 필드를 수집할 수 있음을 확인
- **서비스명 선정**
    - 워커힐 브랜드 철학 `Inspiring Sense & Touching Place`를 참고해 서비스명을 `SensePlace`로 선정
    - 고객 경험과 목소리인 Sense를 분석해 호텔 공간과 운영 현장인 Place의 개선으로 연결한다는 의미 정의
    - 슬로건을 `고객의 경험을 감지하고, 호텔의 변화를 만드는 AI 운영 인사이트 플랫폼`으로 정리
- **다음 작업 정리**
    - 리뷰·운영데이터 스키마 확정 및 문제 유형별 운영지표 매핑
    - 체크인 지연 대표 시나리오 구현
    - 관리자 리포트 화면·출력 항목 설계 및 8월 6일 중간발표용 베이스라인 개발
