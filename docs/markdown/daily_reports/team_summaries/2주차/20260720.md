## 팀 공통 요약

- 문서·요구사항·WBS와 주차별 팀 일일·주간보고 체계를 정비하고 팀 통합 시 보고 자동 집계·검증 절차를 추가했으며 공용 Baseline, 서비스 운영 흐름, SensePlace 방향과 호텔 B2B SaaS 모델을 구체화했다.
- 멘토링에서 Git 활용 방식과 서비스 흐름에 대한 조언을 듣고 이를 바탕으로 오전 팀 회의를 진행했다.
- Git은 팀원별 개인 branch에서 작업한 뒤 `dev`에서 통합 검증하고, 검증이 끝난 최종 버전만 `main`에 반영하기로 했다.
- 서비스 사용자를 대고객이 아닌 호텔 관리자·현업 실무자로 확정하고 비즈니스 모델과 서비스 흐름을 구체화했으며, 중간발표 전 핵심 시나리오 Baseline 완성을 목표로 WBS를 조정하고 사용자 여정·요구사항 기반 화면설계를 진행했다.
- 단순 VOC·리뷰 감성분류를 넘어 외부 리뷰와 내부 운영 데이터를 교차 분석해 문제의 원인 후보와 개선안을 제시하는 서비스로 방향을 정리했다.
