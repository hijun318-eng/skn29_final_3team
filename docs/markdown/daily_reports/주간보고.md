# 주간보고

> 팀원별 일일보고를 주차별로 통합합니다. 유사 내용은 합쳐 요약하고 새로 추가하거나 수정하는 각 주차 보고 블록은 40줄 이내로 유지합니다.

## 20260720 ~ 20260721 (2주차)

작성자: 3팀

- 문서·WBS 체계 정리
  - 문서 분류·파일명 규칙·보호 폴더·참고 기준을 정비하고, WBS 단일 기준을 `02_WBS.md`로 전환해 Mermaid 일정과 갱신 규칙을 적용
  - 요구사항 62건·산출물 21건·실행 태스크 63건을 대조하고 담당 실명화·분류 체계를 반영
- 기획·화면·Baseline 기준 확정
  - 프로젝트 기획서를 요구사항·WBS·조사 자료와 대조해 7개 목차 기준으로 최신화하고, SensePlace 최신 방향의 전량 합성 데이터·2개 서비스 경로·Django/FastAPI 경계·Baseline Gate를 통합
  - 서비스명 SensePlace, 기능 A/B, Django 세션 인증, 3개 역할, 전량 합성 데이터, 8개 보장 질문, 반례 세트 v2 21건, 5회 최종 재현 기준을 네 기획 문서에 동기화했으며 화면설계서는 수정하지 않음
  - 기존 요구사항 ID 62건을 유지하면서 Baseline 43건·독립 실험 8건·보류·승인 후 후보 11건으로 재분류하고, 단일 호텔·전량 합성 데이터 전제에 맞춰 세부 요구사항과 WBS를 재검토
  - 정식 온톨로지는 경량 시맨틱 카탈로그로 조정하고, text-to-SQL은 지원 지표·권한·read-only 검증 범위로 제한하며 군집은 합성 VOC 독립 실험으로 분리
  - 사용자 파일 업로드·대용량 처리·외부 채널 전달은 팀 합의 전까지 보류하고, 규칙 기반 후속 질문·드릴다운은 선택 UX 후보로 정리
  - PM 박준희, A 프론트·UX 송민지, B 백엔드·통합 김재홍, C 데이터·DB 정승, D AI·Agent 윤대성으로 역할을 명시하고, 정승이 발표 전반을 맡으며 PM은 B·D의 scenario·수용 기준·Gate 조율을 지원하도록 재배분
  - 별도 기술 계약 산출물은 기획 단계 범위에서 제외하고, 데이터 항목·시간 단위·권한·8 intent·평가 분리·LLM 모델의 최소 합의만 요구사항과 WBS 선행조건으로 유지
- 서비스·데이터·개발환경 구체화
  - 이상징후→근거→원인 후보→담당자 조치→효과 확인→주간 리포트 흐름, 호텔 B2B SaaS 수익모델, React+Tremor 로컬 개발환경·`frontend/` 제외 설정을 정리
  - `SensePlace` 서비스명과 슬로건을 선정하고, 프로젝트 데이터는 전량 합성으로 제한하며 근거 기반 AI와 관리자 승인 원칙을 명시
- 다음 개발 작업
  - 팀원 검토로 서비스 범위·사용자 흐름·데이터 원칙·역할·Baseline 완료 조건과 미결정 항목을 먼저 확정한 뒤 구현에 착수
