# 문서 관리 규칙

## 기본 구조

`docs/` 바로 아래에는 다음 두 폴더만 사용한다.

```text
docs/
├─ markdown/
└─ deliverables/
   └─ templates/
```

- `markdown/`: `docs/`에서 관리하는 모든 Markdown 문서와 기존 협업·보고 하위 폴더
- `deliverables/`: 제출용 문서, 이미지, 영상, 모델 및 소스 묶음
- `deliverables/templates/`: 제공받은 원본 양식. 원본 파일명은 유지한다.
- `서비스흐름도.png`: 공식 산출물이 아닌 서비스 흐름도 초안으로 `docs/` 바로 아래에 둔다.

## 공식 산출물 파일명

`docs/deliverables/`에 저장하는 공식 산출물만 `두 자리 번호_문서이름_29기_3팀.확장자` 형식을 사용한다.

예시:

```text
01_요구사항정의서_29기_3팀.xlsx
02_WBS_29기_3팀.xlsx
03_프로젝트기획서_29기_3팀.docx
```

- 번호는 `최종_프로젝트_산출물_및_전체_일정.md`의 마감일 순서를 따른다.
- `docs/markdown/` 문서는 이 파일명 규칙을 적용하지 않고 기존의 의미 있는 문서 이름을 유지한다.
- 개인 작업본과 조사 문서는 공식 제출본으로 오해되지 않도록 산출물 이름 규칙을 강제하지 않는다.
- 작업 중 버전은 필요할 때만 파일명 끝에 `_v01`, `_v02`를 붙이고 최종 제출본에서는 제거한다.

## 변경 시 점검

1. 파일 종류에 맞는 폴더인지 확인한다.
2. 공식 산출물이면 일정 문서에서 번호를 확인한다.
3. 기존 파일을 이동하거나 이름을 바꾸면 관련 Markdown 링크와 프로젝트 지침의 경로도 갱신한다.
4. `docs/` 바로 아래에 임시 폴더나 개별 파일을 추가하지 않는다.
