## 산출물별 작성 요구사항

### 1. 요구사항 정의서

- 사용자 시나리오를 바탕으로 기능적·비기능적 요구사항 작성
- 요구사항 우선순위와 예외사항 명시
- 테스트 및 화면 설계와의 연계성 확보
- 문서 변경 이력 관리

### 2. WBS

- GitHub Projects 화면을 글자가 보이도록 캡처하거나 시트/엑셀 파일로 제출
- 두 방식 중 하나만 제출하면 됨
- 업무, 담당, 일정, 의존성, 진행 상태와 완료 기준을 추적할 수 있도록 구성

### 3. 프로젝트 기획서

- 별도 상세 설명은 시트에 기재되어 있지 않음
- 제공 양식의 주요 문서 요소를 포함하되 다른 서식 사용 가능
- 문제, 사용자, 목표, 범위, 핵심 기능, 기대 효과와 기술 방향을 일관되게 정리

### 4. 수집 데이터 보고서

- 수집 데이터의 종류, 규모, 출처 명시
- 프로젝트 목표에 데이터를 어떻게 활용할지 설명
- 개인정보 및 저작권 문제 점검
- 데이터 탐색·필터링 구현 결과 포함

### 5. 데이터베이스/저장소 설계 문서

- 관계형 DB 및 비정형 저장소의 목적 정의
- 테이블·엔터티 관계와 데이터 입출력 흐름 설계
- 원문, 메모리 등 비정형 데이터 저장 방안 포함
- 보안 및 데이터 무결성 관리 방안 포함

### 6. 데이터 전처리 결과서

- 결측치, 이상치, 중복치 처리 기준과 과정 기록
- 원천 데이터가 어떻게 정제·가공되었는지 설명
- 처리 결과가 모델 학습 또는 에이전트 연동에 적합함을 증명

### 7. 머신러닝/딥러닝 학습결과서

- 모델 선정 근거와 하이퍼파라미터 등 학습 환경 작성
- **2개 이상 모델**의 성능 비교
- 최종 모델 선정 사유 명시
- 과적합 여부와 개선 방향 기록

### 8. 학습한 ML/DL 모델

- 실제 구현 모델의 구조와 입출력값 설명
- 모델이 의존하는 전처리 과정 명시
- 모델 저장·불러오기 방법 작성
- 서비스 연계 가능성 설명

### 9. 벡터DB/GraphDB 구축 결과서

- RAG 등을 위한 DB 도입 목적 설명
- 벡터DB 또는 GraphDB의 스키마 구조 작성
- 검색 정확도와 응답 속도 검증 결과 포함
- 데이터 갱신·삭제 등 운영 파이프라인 작성

### 10. AI 시스템 아키텍처

- 서비스 전체 AI 구조와 데이터 흐름을 시각화

## 3팀 관점의 우선순위

3팀의 주제는 **대화형 데이터 기반 분석 에이전트 및 자동 리포팅 플랫폼**이다. 따라서 다음 요소가 산출물 전체에서 일관되게 연결되어야 한다.

1. 사용자 질문과 비즈니스 의사결정 시나리오
2. 분석 대상 데이터와 데이터베이스 구조
3. 자연어 질문의 의도 분석과 데이터 조회 방식
4. 분석·예측 모델의 선정 근거와 평가 결과
5. 멀티 에이전트의 역할, 라우팅 및 실패 대응
6. 답변의 근거 데이터와 자동 리포트 생성 과정
7. 웹서비스 배포, 통합 테스트 및 시연

## 표기 충돌 및 확인 필요 사항

두 탭 사이에 다음과 같은 차이가 있어 담당자 또는 강사 확인이 필요하다.

- **자체 sLLM:** 산출물 안내에는 `3, 4번 주제` 대상으로 표시되지만 대시보드에는 `(필요시)`라고 표시됨
- **웹 애플리케이션:** 산출물 안내에는 8주차로 분류되지만 대시보드 마감은 08/28(7주차)
- **LLM 활용 소프트웨어와 멀티 에이전트 테스트 보고서:** 두 탭에서 명칭과 설명의 대응이 완전히 일치하지 않음

마감일은 우선 `전체 일정 대시보드`의 날짜를 기준으로 관리하되, 위 항목은 프로젝트 초기에 확인하는 것이 안전하다.
